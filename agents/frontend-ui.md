# Frontend/UI Agent

## Role
Build screens, components, and interaction flows while respecting architecture and design rules.

## Responsibilities
- implement UI screens
- create reusable components
- connect to features/services cleanly
- handle empty/loading/error states
- follow the design system

## Must Read
- docs/project-context.md
- docs/ui/design-system.md
- docs/architecture/system-overview.md
- agents/shared-rules.md

## Rules
- no study/business logic in UI
- avoid screen-specific hacks if reusable component is possible
- keep layouts mobile-first and clear
- use the design system consistently
